3 Namespace chapter4_lists

3.1 Types

  • myList (A : Type): custom list type with constructors nil and cons.

3.2 Function Summary

Function Signature Description
list_string myList Nat → String Serializes a Nat list into a concatenated string of element values.
list_length {A : Type} → myList A → Nat Computes the length of a list.
nth_list {A : Type} → myList A → Nat → Option A Returns the element at index i if present, otherwise none.
fold_list (mu : A → (B → B)) → (b : B) → myList A → B Right fold over a list using step function mu and base b.
map_list {A B : Type} → (A → B) → (myList A → myList B) Applies a function to every element of a list.

prod_list myList Nat → Nat Computes the product of all Nat elements in a list.
concat_list {A : Type} → myList A → myList A → myList A Appends the second list onto the first parameterized prefix behavior in this definition order.
flatten_list {A : Type} → myList (myList A) → myList A Flattens a list of lists into a single list.
reverse_list {A : Type} → myList A → myList A Reverses a list recursively using concat_list.

4 Namespace chapter4_integers

4.1 Function Summary

Function Signature Description
addNaturalToZ myN → myZ → myZ Adds a positive natural amount to an integer by repeated successor.
subtractNaturalFromZ myN → myZ → myZ Subtracts a positive natural amount from an integer by repeated predecessor.
myAddZ myZ → myZ → myZ Integer addition by case analysis on the second argument.
multNaturalWithZ myZ → myN → myZ Multiplies an integer by a positive natural through repeated addition.
myMultZ myZ → myZ → myZ Integer multiplication by reducing to natural multiplication and sign handling.

5 Notes

  • The booleans section mostly develops constructive logic transformations using myNegType and myEquiv from chapter3.
  • The list type uses a custom inductive encoding where cons takes an element and then a list constructor argument.
  • Integer operations rely on predecessor and successor machinery defined in chapter3.
